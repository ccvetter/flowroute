08 — Design principles

01

Data density without noise

Warehouse ops require high-information displays. Every pixel earns its place — no decorative chrome. Use monospaced fonts for all scannable data (IDs, timestamps, quantities).

02

Amber = action needed

Amber is reserved strictly for items requiring human attention. Never use it decoratively. Staff should be conditioned to act when they see amber.

03

Glanceable at distance

Fulfillment staff may view screens from 1–3 meters away. Status badges and key metrics use minimum 14px with strong contrast. Touch targets ≥ 44px.

04

Exception-first visibility

Problems surface to the top automatically. The default sort on every list view is: exceptions → at-risk → on-track. Never bury bad news in pagination.

05

Keyboard & scanner first

All critical actions reachable by keyboard shortcut or barcode scan. Mouse is secondary. Staff should be able to process an order without touching a mouse.
